## VPS Security Hardening (1 May 2026)

### Issues Found & Fixed
- **Root SSH login** (`PermitRootLogin yes`) — ADVISED but NOT fixed pending user approval
- **Firewall**: iptables was ACCEPT all. Fixed: firewalld active, ports 22/80/443 allowed, ports 5001-5006 blocked from external
- **SELinux/AppArmor**: Disabled
- **Fail2ban**: Not available in OpenCloudOS 9.4 repos (EPEL not found). Created custom iptables-based SSH brute-force protection (cron job `/root/ssh_ban.sh`)
- **Port 5001-5006**: Were bound to 0.0.0.0. Fixed: all apps bound to 127.0.0.1 only
- **Nginx**: `server_tokens off` to hide version
- **Security headers** added: HSTS, X-Frame-Options, X-Content-Type-Options, X-XSS-Protection, Referrer-Policy, Permissions-Policy
- **Rate limiting** added: 30 req/s with burst 50 via nginx

## Sirup Bulukumba Production Readiness (1 May 2026 PM)

### Flask → Gunicorn Migration
- **Before**: `app.run(host='0.0.0.0', port=5005)` — single-threaded Flask dev server
- **After**: Gunicorn with 4 workers, systemd service, auto-restart
- **Memory**: ~332MB for 4 workers (was ~100MB with dev server)
- **Capacity**: From ~10 concurrent users → ~50+
- **Debug mode**: OFF (was ON — security risk)

### Geoportal (port 5006) - Also Migrated
- Gunicorn 2 workers, systemd service
- Debug mode disabled
- Bound to 127.0.0.1

### Template Optimization
- **Before**: 98KB template, 1548 lines, all data server-rendered via Jinja2, page load 1.5s
- **After**: 13KB lightweight skeleton, data loads via AJAX (`/api/stats` + `/api/table`), page load 48ms (30x faster, 97% smaller)
- `/api/stats` endpoint created for global stats (cached, 8ms response)
- `/api/table` already existed for paginated table data
- Charts rendered client-side via Chart.js CDN
- Auto-refresh every 5 minutes via JS interval
