## Callunk's Rules for All Future Apps

1. **Global style untuk semua komponen** — textbox, dropdown, textarea, label, tombol, table, badge, stat card, modal — semua harus ikut satu design system. Gunakan CSS modular terpisah (contoh: `static/css/mk-style.css` dengan class `mk-input`, `mk-select`, `mk-btn`, `mk-label`, dll).

2. **Setiap searchbox harus suggest-input** — semua input pencarian di halaman list maupun form harus pakai `.suggest-input` dengan `data-type` sesuai sumber data. Dilengkapi arrow ▾ (show all) dan clear ✕. Suggestion dari API `/api/suggest?type=...&q=...`.

3. **Setiap textbox yang bisa diinput (terikat data) harus punya:**
   - Tombol **✕** (clear) — muncul kalo ada nilai
   - **Autocomplete** — ketika diketik, muncul suggestion dari server
   - **Arrow ▾** — klik untuk show all data (dropdown mode)
   - **Tambah Baru** — selalu muncul di bagian bawah suggestion, kalo data kosong atau user mau nambah data baru
   - Navigasi keyboard: ↑↓ pilih, Enter pilih, Escape tutup

4. Pasien search di form resep pake handler khusus via `/api/pasien/search`, bukan suggest API umum.
